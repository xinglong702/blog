# 读书笔记【Js红宝书】原始值包装类型

为了方便操作原始值，ECMAScript提供了3种特殊的引用类型：String、Number、Boolean。这些类型既具有其他一般引用类型的特点，也具有与各自原始类型对应的特殊行为【重写了某些方法（比如valueOf()和）】。

每当用到某个原始值的方法或属性时【原始值是没有属性和方法的！！！】，后台都会创建一个相应的原始值包装类型的对象。

  • 来看下面的例子：
let str1 = '我是原始值字面量str1'
let str2 = str1.substring(2)	// 在原始值str1上使用了对象才有的方法，后台肯定帮我们做了些什么？
1
2
  • 既然能够在原始值上使用方法，那能不能添加属性呢？且继续看下面代码：
str1.newProp = '我是str1上的新属性'

console.log(str1.newProp)	// undefined
// 如果可以的话，那控制台的输出应该是：我是str1上的新属性
// 但实际上输出为：undefined
1
2
3
4
5
  • 为什么又不行了呢？为什么输出的是undefined，而不是报错呢？

原因在于： 在把原始值当作对象使用时，后台会执行以下3步：

1）创建一个String类型的实例

2）调用实例上的特定方法

3）销毁实例

相当于执行了以下代码：

let str1 = new String('我是原始值字面量str1')	// 1）创建一个String类型的实例
let str2 = str1.substring(2)				// 2）调用实例上的特定方法
str1 = null									// 3）销毁实例
1
2
3
  • 现在就能解释上面的问题了：
str1.newProp = '我是str1上的新属性'
/* 这个新属性在这一行代码执行时，确实添加成功了（3步默认在后台执行），
	但第3步这个实例就被销毁了，也就是说这行代码执行完后就不存在包装的实例对象了！！！
*/

console.log(str1.newProp)
/* 同理，这里后台也执行了3步，但没有指定newProp的值，自然就是Undefined了！！！
	执行完这一行语句后，该包装的实例对象也会被销毁。
	这一句代码其实和上一句代码没有什么关系！！！
*/

  • 注意：使用new调用原始值包装类型的构造函数，与调用同名的转型函数并不一样。请看下面的例子：
let s1 = '1'				// 原始值
let s2 = new String('1')	// new一个原始值包装对象			// 三个值看起来都是‘1’
let s3  = String(1)			// 转型函数，将1转成‘1’
console.log(s1 == s2)   	// true 			
console.log(s1 === s2)  	// false
console.log(s1 == s3)   	// true
console.log(s1 === s3)  	// true
console.log(s2 == s3)   	// true
console.log(s2 === s3)  	// false

let s4 = new String('1')
console.log(s2 == s4)   	// false
console.log(s2 === s4)  	// false
