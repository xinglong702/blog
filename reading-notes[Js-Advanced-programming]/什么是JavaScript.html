# 第一章——什么是JavaScript

# 1.1 简短的历史回顾

  • 为何会出现JavaScript这门语言？

    • 在JavaScript问世之前，每次表单验证都需要用户多次与服务器进行通信(由服务器端的语言来处理)。想象一下，你填完注册表单，点击提交按钮，在等待30s后，看到一条消息，告诉你用户名不符合规则，在你填完之后点击提交，又过了30s，告诉你用户名已被占用。。。。暴躁一点的老哥就要砸电脑了把😄
    • 那我们能不能在客户端就完成这些简单的工作呢？ 1995年，网景公司（Netscape——一家十分悲催的公司）一位名叫Brendan Eich的工程师，开始为即将发布的Netscape Navigator 2【火狐的前生，干不过IE被迫捐出😭】开发一个叫Mocha【后来改名叫LiveScript，再后来为了搭上媒体当时热烈炒作的Java的顺风车，又改名为JavaScript，所以两者其实关系并不大】的脚本语言，这门语言能够在客户端完成一些简单的工作。如今，JavaScript的应用也不再局限于数据验证，而是渗透到浏览器窗口及其内容的方方面面。

  • 网景公司和微软之间的竞争

    • 一开始，网景公司凭借着Javascript1.0的成功，稳坐浏览器市场第一把交椅，微软一看不对劲，就紧跟着网景公司的脚步，发布了IE3和自己名为JScript的Javascript实现【这是网景公司永远的痛😭，微软不讲武德呀】
    • 微软的JScript出现意味着市场上存在着两个版本的JavaScript，如果任由其发展下去，前端工程师以后岂不是要面向浏览器编程了😆。随着业界担忧日甚，javaScript终于踏上了标准化的征程。
    • 网景公司一看打不过微软了，就舍生取义，先发制人，主动将Javascript1.1献给欧洲计算机制造商协会（Ecma），第39技术委员会（TC39）花了数月的时间打造出ECMA-262，也就是ECMAScript这个新的脚本语言标准。
    • 自此以后，各家浏览器均以ECMAScript作为自己实现JavaScript的标准，虽然具体实现各有不同。

  • 时间表如下：

    年份 事件
    1995 网景公司开发了Javascript
    1996 微软发布了和Javascript兼容的JScript【速度之快，令人震惊！】
    1997 ECMAScript第1版（ECMA-262）
    1998年 ECMAScript第2版
    1998 DOM Level1的制定
    1998 新型语言DHTML(D是动态的意思)登场
    1999 ECMAScript第3版
    2000 DOM Level2的制定
    2004 DOM Level3的制定
    2005 新型语言AJAX登场
    2009 ECMAScript第5版【为啥没有第4版呢？因为第4版几乎是定义了一门新语言，包括强类型变量、真正的类和经典的继承（这和原来的基于原型的类的实现完全不同），最后被废弃了】
    2009 新型语言HTML5登场
    2015 ECMAScript第6版，也就是大名鼎鼎的ES6（ES2015）。ES6正式支持了类、模块、迭代器、生成器、箭头函数、Promise、代理、反射和众多新的数据类型。
    2016 ES7。少量改动。
    2017 ES8。增加了异步函数（async/await）
    2018 ES9
    2019 ES10
    2020 ES11

# 1.2 JavaScript实现

  • 完整的JavaScript实现包括以下几个部分：
    • 核心（ECMAScript）
    • 文档对象模型(DOM)
    • 浏览器对象模型（BOM）

# 1.2.1 ECMAScript

ECMAScript，即ECMA-262定义的语言，并不局限于Web浏览器【宿主环境还有服务端Javascript平台Node.js】，事实上，这门语言没有输入和输出之类的方法。

# 1.2.2 DOM

文档对象模型（Document Object Model）是一个应用编程接口（API）。DOM将整个页面抽象为一组分层节点。每个组成部分都是一种节点，包含不同的数据。比如下面的HTML页面：

<html>
<head>
    <meta/>
    <title>Sample Page</title>
</head>
<body>
    <p>Hello Js</p>
</body>
</html>
